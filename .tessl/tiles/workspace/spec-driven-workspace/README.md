# workspace/spec-driven-workspace

Tile local para `Spec As Source` en el workspace root.

## Uso

Incluye `use spec-driven development` en tu prompt y trabaja desde la spec del root.

## Principios

- las specs viven en `specs/**`
- los `targets` enrutan el trabajo hacia submodulos
- `.flow/**` guarda estado operativo y reportes
- no se implementa codigo sin spec aprobada
